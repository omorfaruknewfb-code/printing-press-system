# Printing Press Management System - Complete Feature Documentation

## সফটওয়্যার সম্পর্কে সারসংক্ষেপ (Overview)

এটি একটি **Next.js 14** ভিত্তিক প্রিন্টিং প্রেস ম্যানেজমেন্ট সিস্টেম যা **TypeScript**, **Prisma ORM**, **PostgreSQL (Supabase)**, এবং **NextAuth** ব্যবহার করে তৈরি করা হয়েছে। এটি প্রিন্টিং প্রেস ব্যবসার জন্য সম্পূর্ণ সমাধান প্রদান করে।

## ২. ডাটাবেস স্কিমা (Database Schema)

### মূল মডেলসমূহ (Core Models)

#### User (ব্যবহারকারী)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `name` (String) - ব্যবহারকারীর নাম
  - `email` (String, Unique) - ইমেইল ঠিকানা
  - `passwordHash` (String) - হ্যাশড পাসওয়ার্ড
  - `role` (Enum: ADMIN, STAFF) - ব্যবহারকারীর ভূমিকা
  - `isActive` (Boolean) - একাউন্ট সক্রিয় কিনা
  - `avatarUrl` (String, Optional) - প্রোফাইল ছবি
  - `commissionRate` (Decimal) - কমিশন হার (শতাংশে)
  - `createdAt`, `updatedAt` (DateTime) - টাইমস্ট্যাম্প

#### Customer (গ্রাহক)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `name` (String) - গ্রাহকের নাম
  - `mobile` (String, Unique) - মোবাইল নম্বর
  - `address` (String, Optional) - ঠিকানা
  - `totalOrders` (Int) - মোট অর্ডার সংখ্যা
  - `totalBill` (Decimal) - মোট বিল
  - `totalPaid` (Decimal) - মোট পরিশোধিত
  - `totalDue` (Decimal) - মোট বাকি
  - `createdAt`, `updatedAt` (DateTime)

#### Order (অর্ডার)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `orderId` (String, Unique) - অর্ডার আইডি (ডিসপ্লে জন্য)
  - `date` (DateTime) - অর্ডার তারিখ
  - `customerId` (String) - গ্রাহক আইডি (Foreign Key)
  - `totalBill` (Decimal) - মোট বিল
  - `advance` (Decimal) - অগ্রিম পেমেন্ট
  - `due` (Decimal) - বাকি টাকা
  - `totalCosting` (Decimal) - মোট খরচ
  - `costingSet` (Boolean) - খরচ সেট করা হয়েছে কিনা
  - `deliveryDate` (DateTime) - ডেলিভারি তারিখ
  - `status` (Enum: PENDING, CONFIRMED, READY_FOR_DELIVERY, DELIVERED, CANCELLED)
  - `orderTakenById` (String) - অর্ডার নেওয়া স্টাফের আইডি
  - `assignedToId` (String, Optional) - অ্যাসাইন করা স্টাফের আইডি
  - `createdAt`, `updatedAt` (DateTime)

#### OrderItem (অর্ডার আইটেম)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `orderId` (String) - অর্ডার আইডি (Foreign Key)
  - `itemName` (String) - আইটেমের নাম
  - `description` (String, Optional) - বিবরণ
  - `quantity` (Int) - পরিমাণ
  - `unitPrice` (Decimal) - একক মূল্য
  - `lineTotal` (Decimal) - লাইন টোটাল

#### CustomerPayment (গ্রাহক পেমেন্ট)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `customerId` (String) - গ্রাহক আইডি
  - `orderId` (String, Optional) - অর্ডার আইডি
  - `billAmount` (Decimal) - বিলের পরিমাণ
  - `paidAmount` (Decimal) - পরিশোধিত পরিমাণ
  - `dueAmount` (Decimal) - বাকি পরিমাণ
  - `remarks` (String, Optional) - মন্তব্য
  - `paymentDate` (DateTime) - পেমেন্ট তারিখ
  - `collectedById` (String, Optional) - টাকা নেওয়া স্টাফের আইডি
  - `createdAt`, `updatedAt` (DateTime)

#### Expense (খরচ)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `date` (DateTime) - তারিখ
  - `category` (String) - খরচের ক্যাটাগরি
  - `amount` (Decimal) - পরিমাণ
  - `description` (String, Optional) - বিবরণ
  - `createdById` (String) - যে স্টাফ খরচ এন্ট্রি করেছে
  - `createdAt`, `updatedAt` (DateTime)

#### ExpenseCategory (খরচের ক্যাটাগরি)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `name` (String, Unique) - ক্যাটাগরির নাম
  - `createdAt` (DateTime)

#### StaffWithdrawal (স্টাফ উত্তোলন)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `staffId` (String) - স্টাফ আইডি
  - `type` (Enum: COMMISSION, SALARY, OTHER) - উত্তোলনের ধরন
  - `amount` (Decimal) - পরিমাণ
  - `date` (DateTime) - তারিখ
  - `remarks` (String, Optional) - মন্তব্য
  - `relatedExpenseId` (String, Optional) - সম্পর্কিত খরচ আইডি
  - `editedByAdmin` (Boolean) - অ্যাডমিন এডিট করেছে কিনা
  - `createdAt`, `updatedAt` (DateTime)

#### OrderStatusHistory (অর্ডার স্ট্যাটাস হিস্ট্রি)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `orderId` (String) - অর্ডার আইডি
  - `oldStatus` (Enum, Optional) - পুরানো স্ট্যাটাস
  - `newStatus` (Enum) - নতুন স্ট্যাটাস
  - `changedBy` (String) - যে ব্যবহারকারী পরিবর্তন করেছে
  - `remarks` (String, Optional) - মন্তব্য
  - `changedAt` (DateTime) - পরিবর্তনের সময়

#### ItemMaster (আইটেম মাস্টার)
- **ফিল্ডসমূহ:**
  - `id` (String, CUID) - ইউনিক আইডেন্টিফায়ার
  - `itemName` (String, Unique) - আইটেমের নাম
  - `createdAt`, `updatedAt` (DateTime)

#### Settings (সেটিংস)
- **ফিল্ডসমূহ:**
  - `id` (String, Default: "default") - ইউনিক আইডেন্টিফায়ার
  - `orgName` (String, Default: "Al-Ihsan Printing Press") - প্রতিষ্ঠানের নাম
  - `logoUrl` (String, Optional) - লোগো URL

#### Counter (কাউন্টার)
- **ফিল্ডসমূহ:**
  - `name` (String, ID) - কাউন্টারের নাম
  - `value` (Int, Default: 0) - মান

## ৩. অথেনটিকেশন সিস্টেম (Authentication System)

### ডিফল্ট ক্রেডেনশিয়ালস (Default Credentials)

**অ্যাডমিন অ্যাকাউন্ট:**
- ইমেইল: `admin@alihsan.com`
- পাসওয়ার্ড: `Admin@123`
- ভূমিকা: ADMIN

**স্টাফ অ্যাকাউন্ট:**
- ইমেইল: `staff@alihsan.com`
- পাসওয়ার্ড: `Staff@123`
- ভূমিকা: STAFF

> **গুরুত্বপূর্ণ:** এই ক্রেডেনশিয়ালস `prisma/seed.ts` ফাইলে সেট করা আছে। যদি আপনি পরিবর্তন করতে চান, তবে `.env` ফাইলে `ADMIN_SEED_PASSWORD` এবং `STAFF_SEED_PASSWORD` এনভায়রনমেন্ট ভেরিয়েবল সেট করতে পারেন।

### অথেনটিকেশন ফিচারসমূহ
- **NextAuth.js** ব্যবহার করে ক্রেডেনশিয়ালস প্রোভাইডার
- **JWT Session Strategy** - ৮ ঘন্টা সেশন টাইমআউট
- **bcrypt** দিয়ে পাসওয়ার্ড হ্যাশিং (salt rounds: 10)
- **ভূমিকা ভিত্তিক অ্যাক্সেস কন্ট্রোল** (ADMIN vs STAFF)
- **সক্রিয়/নিষ্ক্রিয় ব্যবহারকারী** চেক
- **কাস্টম লগইন পেজ** (`/login`)
- **মিডলওয়্যার প্রোটেকশন** - সব ড্যাশবোর্ড রুট প্রোটেক্টেড

### পাসওয়ার্ড পরিবর্তন সম্পর্কে তথ্য
- পাসওয়ার্ড পরিবর্তন করতে হলে ডাটাবেসে সরাসরি আপডেট করতে হবে অথবা প্রোফাইল পেজ থেকে পরিবর্তন করতে হবে
- পাসওয়ার্ড হ্যাশ করা থাকে, তাই সরাসরি পাঠানো যায় না
- bcrypt দিয়ে হ্যাশ করতে হবে

### ৪.২ অর্ডার ম্যানেজমেন্ট (Order Management)

#### অর্ডার তৈরি (Create Order):
- **গ্রাহক নির্বাচন:** অটোকমপ্লিট সার্চ (নাম বা মোবাইল দিয়ে)
- **নতুন গ্রাহক তৈরি:** অর্ডার ফর্ম থেকেই
- **আইটেম যোগ:** মাল্টিপল আইটেম যোগ করা যায়
  - আইটেম নাম (অটোকমপ্লিট থেকে ItemMaster)
  - বিবরণ (ঐচ্ছিক)
  - পরিমাণ
  - একক মূল্য
  - অটোমেটিক লাইন টোটাল ক্যালকুলেশন
- **অর্ডার তথ্য:**
  - অর্ডার তারিখ (ডিফল্ট: আজ)
  - ডেলিভারি তারিখ
  - অগ্রিম পেমেন্ট
  - অটোমেটিক বাকি ক্যালকুলেশন
- **স্টাফ অ্যাসাইনমেন্ট:**
  - অর্ডার নেওয়া স্টাফ (লগইন করা ব্যবহারকারী)
  - অ্যাসাইন করা স্টাফ (ঐচ্ছিক)
- **অটোমেটিক অর্ডার আইডি জেনারেশন**

#### অর্ডার লিস্ট ও ফিল্টারিং:
- **সার্চ:** অর্ডার আইডি, আইটেম নাম, গ্রাহক নাম, মোবাইল দিয়ে
- **স্ট্যাটাস ফিল্টার:** PENDING, CONFIRMED, READY_FOR_DELIVERY, DELIVERED, CANCELLED, ALL
- **তারিখ ফিল্টার:** তারিখ রেঞ্জ দিয়ে
- **বাকি ফিল্টার:** শুধু বাকি আছে এমন অর্ডার দেখানো
- **সর্টিং:** সর্বশেষ অর্ডার আগে দেখানো

#### অর্ডার ডিটেইল পেজ:
- **সম্পূর্ণ অর্ডার তথ্য:** গ্রাহক, আইটেমসমূহ, পেমেন্ট, স্ট্যাটাস
- **কস্টিং এন্ট্রি:** মোট খরচ সেট করা
- **স্ট্যাটাস পরিবর্তন:**
  - PENDING → CONFIRMED → READY_FOR_DELIVERY → DELIVERED
  - যেকোনো স্ট্যাটাসে CANCELLED করা যায়
  - স্ট্যাটাস হিস্ট্রি ট্র্যাক করা হয়
- **পেমেন্ট কালেকশন:** গ্রাহক থেকে টাকা নেওয়া
- **ডেলিভারি:** অর্ডার ডেলিভারড মার্ক করা
- **প্রিন্ট স্লিপ:** ক্লায়েন্ট কপি প্রিন্ট করা
- **অর্ডার এডিট:** অর্ডার তথ্য পরিবর্তন করা (যদি ডেলিভারড না হয়)

#### পেন্ডিং ওয়ার্কস (Pending Works):
- **পেন্ডিং অর্ডার লিস্ট:** ডেলিভারি তারিখ অনুযায়ী সর্ট করা
- **কুইক অ্যাকশন:** স্ট্যাটাস পরিবর্তন, কস্টিং এন্ট্রি, পেমেন্ট কালেকশন

### ৪.৩ গ্রাহক ম্যানেজমেন্ট (Customer Management)

#### গ্রাহক লিস্ট:

### ৪.৪ খরচ ম্যানেজমেন্ট (Expense Management)

#### খরচ এন্ট্রি:
- **তারিখ:** খরচের তারিখ
- **ক্যাটাগরি:** ড্রপডাউন থেকে নির্বাচন
  - ডিফল্ট ক্যাটাগরিস: Paper Purchase, Ink Purchase, Electricity, Transport, Miscellaneous, Staff Payment
  - নতুন ক্যাটাগরি যোগ করা যায়
- **পরিমাণ:** টাকার পরিমাণ
- **বিবরণ:** খরচের বিবরণ (ঐচ্ছিক)

#### খরচ ক্যাটাগরি ম্যানেজমেন্ট:
- **ক্যাটাগরি তালিকা:** সব ক্যাটাগরি দেখা
- **নতুন ক্যাটাগরি যোগ:** কাস্টম ক্যাটাগরি তৈরি
- **ক্যাটাগরি ডিলিট:** ব্যবহার করা হচ্ছে এমন ক্যাটাগরি ডিলিট করা যায় না

#### খরচ লিস্ট ও ফিল্টারিং:
- **ক্যাটাগরি ফিল্টার:** নির্দিষ্ট ক্যাটাগরির খরচ দেখা
- **তারিখ ফিল্টার:** তারিখ রেঞ্জ দিয়ে
- **মোট খরচ:** ফিল্টার অনুযায়ী মোট খরচ দেখানো

### ৪.৫ রিপোর্টস (Reports)

#### মাসিক ফাইন্যানশিয়াল রিপোর্ট:
- **মাসিক রেভিনিউ:** প্রতি মাসের মোট বিল
- **মাসিক খরচ:** প্রতি মাসের মোট খরচ
- **মাসিক প্রফিট:** রেভিনিউ - খরচ
- **চার্ট:** ৬ মাসের ডেটা ভিজুয়ালাইজেশন

#### দৈনিক ফাইন্যানশিয়াল রিপোর্ট (বর্তমান মাস):
- **দৈনিক রেভিনিউ:** প্রতি দিনের মোট বিল
- **দৈনিক খরচ:** প্রতি দিনের মোট খরচ
- **চার্ট:** বর্তমান মাসের প্রতি দিনের ডেটা

#### প্রফিট সামারি:
- **রেভিনিউ:** কস্টিং সেট করা অর্ডারের মোট বিল
- **কস্টিং:** মোট খরচ (প্রোডাকশন)
- **গ্রস প্রফিট:** রেভিনিউ - কস্টিং
- **টোটাল খরচ:** সব ধরনের খরচ (স্টাফ পেমেন্ট সহ)
- **নেট প্রফিট:** গ্রস প্রফিট - টোটাল খরচ

#### টপ আইটেমস:
- **টপ ৫ আইটেম:** মোট পরিমাণ অনুযায়ী সবচেয়ে বেশি বিক্রি হওয়া আইটেম

#### টপ কাস্টমারস:
- **অর্ডার সংখ্যা অনুযায়ী:** সবচেয়ে বেশি অর্ডার দেওয়া গ্রাহক
- **মোট বিল অনুযায়ী:** সবচেয়ে বেশি টাকার অর্ডার দেওয়া গ্রাহক

#### স্টাফ পারফরম্যান্স:
- **প্রতি স্টাফের পারফরম্যান্স:**
  - মোট অর্ডার নেওয়া (বর্তমান মাস)
  - ডেলিভারড অর্ডার (বর্তমান মাস)
  - কালেকশন (বর্তমান মাস)

### ৪.৬ ব্যবহারকারী ম্যানেজমেন্ট (User Management)

#### ব্যবহারকারী লিস্ট (শুধুমাত্র অ্যাডমিন):
- **সব ব্যবহারকারীর তালিকা**
- **তথ্য:** নাম, ইমেইল, ভূমিকা, সক্রিয়/নিষ্ক্রিয়, কমিশন হার, তৈরির তারিখ
- **ব্যবহারকারী তৈরি:** নতুন স্টাফ যোগ করা
- **ব্যবহারকারী এডিট:** নাম, ইমেইল, ভূমিকা, কমিশন হার পরিবর্তন
- **ব্যবহারকারী ডিলিট/ডিএক্টিভেট:** একাউন্ট বন্ধ করা

#### স্টাফ উইথড্রয়াল হিস্ট্রি:
- **সব উইথড্রয়ালের তালিকা**
- **ফিল্টারিং:** স্টাফ, ধরন, তারিখ দিয়ে
- **এডিট:** অ্যাডমিন কর্তৃক উইথড্রয়াল এডিট করা যায়

### ৪.৭ স্টাফ কমিশন সিস্টেম (Staff Commission System)

#### কমিশন ক্যালকুলেশন:
- **ফর্মুলা:** কমিশন = (মোট বিল - মোট কস্টিং) × কমিশন হার / ১০০
- **শর্ত:** শুধুমাত্র কস্টিং সেট করা অর্ডারের জন্য
- **সময়কাল:** বর্তমান মাস

#### কমিশন উইথড্রয়াল:
- **ধরন:** COMMISSION, SALARY, OTHER
- **পরিমাণ:** উইথড্রয়াল পরিমাণ
- **মন্তব্য:** ঐচ্ছিক নোট
- **ট্র্যাকিং:** কত কমিশন আর্ন হয়েছে, কত উইথড্র করা হয়েছে, কত বাকি আছে

#### মাই কালেকশন (My Collection) - স্টাফ ফিচার:
- **নিজের অর্ডার লিস্ট:** শুধুমাত্র নিজের অর্ডার
- **কস্টিং এন্ট্রি:** নিজের অর্ডারের কস্টিং সেট করা
- **পেমেন্ট কালেকশন:** নিজের অর্ডার থেকে টাকা নেওয়া
- **কমিশন উইথড্রয়াল:** কমিশন উত্তোলন করা
- **উইথড্রয়াল হিস্ট্রি:** নিজের উইথড্রয়ালের তালিকা

### ৪.৮ সেটিংস (Settings)

#### অর্গানাইজেশন সেটিংস:
- **প্রতিষ্ঠানের নাম:** অ্যাপে দেখানো নাম
- **লোগো:** প্রতিষ্ঠানের লোগো আপলোড
- **গ্লোবাল অ্যাপ্লিকেশন:** সব জায়গায় এই সেটিংস প্রয়োগ হয়

### ৪.৯ প্রোফাইল (Profile)

#### ব্যক্তিগত প্রোফাইল:
- **প্রোফাইল দেখা:** নাম, ইমেইল, ভূমিকা
- **পাসওয়ার্ড পরিবর্তন:** বর্তমান পাসওয়ার্ড দিয়ে নতুন পাসওয়ার্ড সেট করা
- **অ্যাভাতার:** প্রোফাইল ছবি আপলোড (ঐচ্ছিক)

### ৪.১০ প্রিন্ট স্লিপ (Print Slip)

#### অর্ডার প্রিন্ট:
- **ক্লায়েন্ট কপি:** গ্রাহকের জন্য প্রিন্ট কপি
- **তথ্য:** অর্ডার আইডি, গ্রাহক তথ্য, আইটেমসমূহ, মোট বিল, অগ্রিম, বাকি, ডেলিভারি তারিখ
- **প্রিন্ট-ফ্রেন্ডলি:** সুন্দর ফরম্যাটে প্রিন্ট হয়

## ৭. এনভায়রনমেন্ট ভেরিয়েবল (Environment Variables)

```env
ADMIN_SEED_PASSWORD="Admin@123"

```

## ৮. ইনস্টলেশন ও রান করার নির্দেশিকা (Installation & Setup Guide)

### প্রয়োজনীয়তা (Prerequisites):
- Node.js 18+ ইনস্টল থাকতে হবে
- npm বা yarn ইনস্টল থাকতে হবে
- PostgreSQL ডাটাবেস (Supabase রিকমেন্ডেড)

### ইনস্টলেশন স্টেপস:

১. **রিপোজিটরি ক্লোন করা:**
```bash
git clone <repository-url>
cd printing-press-system
```

২. **ডিপেন্ডেন্সিস ইনস্টল করা:**
```bash
npm install
```

৩. **এনভায়রনমেন্ট ভেরিয়েবল সেট করা:**
```bash
cp .env.example .env
# .env ফাইল এডিট করে আপনার ডাটাবেস কানেকশন দিন
```

৪. **Prisma সেটআপ:**
```bash
# ডাটাবেস পুশ করা (স্কিমা সিঙ্ক)
npx prisma db push

# সিড ডেটা ইনসার্ট করা
```

## ১০. অথেনটিকেশন সমস্যা সমাধান (Authentication Troubleshooting)

### লগইন সমস্যা হলে করণীয়:

#### সমস্যা ১: ডিফল্ট পাসওয়ার্ড দিয়ে লগইন হচ্ছে না
**সমাধান:**
১. ডাটাবেসে সরাসরি চেক করুন যে সিড রান হয়েছে কিনা
২. আবার সিড রান করুন:
```bash
npm run prisma:seed
```
৩. যদি এনভায়রনমেন্ট ভেরিয়েবল সেট করে থাকেন, তবে সেগুলো চেক করুন

#### সমস্যা ২: পাসওয়ার্ড পরিবর্তনের পর লগইন হচ্ছে না
**সমাধান:**
১. পাসওয়ার্ড হ্যাশ করা হয়েছে তা নিশ্চিত করুন
২. bcrypt দিয়ে হ্যাশ করতে হবে (salt rounds: 10)
৩. ডাটাবেসে `passwordHash` ফিল্ড আপডেট করুন

#### সমস্যা ৩: সেশন এক্সপায়ার হয়ে যাচ্ছে
**সমাধান:**
- সেশন টাইমআউট ৮ ঘন্টা সেট করা আছে
- `src/lib/auth.ts` এ `maxAge` পরিবর্তন করতে পারেন

#### সমস্যা ৪: NEXTAUTH_SECRET সমস্যা
**সমাধান:**
- `.env` ফাইলে `NEXTAUTH_SECRET` সেট করা আছে তা নিশ্চিত করুন
- নতুন সিক্রেট জেনারেট করতে পারেন:
```bash
openssl rand -base64 32
```
